## 📌 1. JPA 연관관계 개요

- JPA란? 자바 애플리케이션에서 관계형 데이터베이스의 데이터를 관리하기 위해 객체와 관계를 매핑하는 API
- 엔티티(Entity): 데이터베이스 테이블에 대응되는 객체로 특정 데이터 모델을 표현
- 연관관계의 의미: JPA 엔티티 간의 관계를 정의하며, 한 엔티티가 다른 엔티티와 어떻게 연결되어 있는지를 설명함
- 이해해야 하는 이유: 애플리케이션 내 데이터를 효율적으로 구성하여 성능 향상, 데이터 무결성, 유지 보수성을 확보하기 위함

## 📌 2. One-to-Many (일대다 연관관계)

부모 측(예: 가족)과 자식 측(예: 구성원)의 관계다

🔹 양방향 One-to-Many

- 전이(Cascade) 방향: 항상 부모 측에서 자식 측으로만 전이를 사용해야 함
- 자식 측(@ManyToOne)에는 전이 타입을 붙이면 안된다
🔹 단방향 One-to-Many

- 자식 측에서 부모를 참조하지 않기 때문에, 외래키를 저장할 수 없어 “별도의 연결 테이블(Link Table)"이 자동 생성된다
- 문제점 1 (비효율성): 연결 테이블 자체의 오버헤드, 2개의 외래키 인덱싱으로 인한 메모리 사용 증가, 3개 테이블 조인으로 인한 쿼리 복잡성 및 성능 저하가 발생한다
- 문제점 2 (INSERT/DELETE 비효율): 새로운 자식(구성원)을 하나 추가할 때, 연결 테이블에 있는 “기존 관계를 모두 삭제(DELETE)한 후, 새로운 자식을 포함하여 전체를 다시 삽입(INSERT)”하는 비효율적인 쿼리가 발생합니다. 이는 부모가 자식의 상태를 정확히 파악하기 제한적이기 때문이다
- 💡 해결 방법 (@JoinColumn): 부모 엔티티에 @JoinColumn을 추가하면 연결 테이블 없이 자식 테이블의 외래키를 직접 제어할 수 있습니다. 단, 여전히 외래키 설정을 위한 추가 UPDATE 쿼리가 발생하여 어느 정도의 성능 저하는 발생한다

## 📌 4. One-to-One (일대일 연관관계) 

🔹 단방향 One-to-One

- 자식 측에 어노테이션이 붙으며, 부모는 자식의 식별자를 알지 못한다
- 자식 정보를 가져오기 위해 조건 탐색을 해야 하므로, 부모가 자식을 필요로 할 때마다 매번 추가 쿼리가 발생하여 성능이 저하될 수 있다
🔹 양방향 One-to-One

- 부모 측에 @MappedBy를 추가하여 설정합니다.
- 문제점: 지연 로딩(LAZY)을 설정하더라도 부모를 조회할 때 자식까지 무조건 SELECT 되는 문제가 있다
💡 해결 방법: @MapsId 사용

- 자식 측에 @MapsId를 추가하여 자식 테이블이 부모 테이블과 기본키(PK)를 공유하도록 설정합니다. (자식 식별자 생성 전략(Strategy)은 제거)
- 부모 엔티티에서는 굳이 양방향 매핑을 할 필요가 없어 구조가 단순화
- 장점 1: 부모 조회 시 자식을 불필요하게 가져오는 부가 쿼리 문제가 해결
- 장점 2: 기본키와 외래키를 모두 인덱싱할 필요가 없어 메모리 사용량이 감소
